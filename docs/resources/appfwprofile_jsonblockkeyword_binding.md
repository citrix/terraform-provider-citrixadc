# Resource: appfwprofile_jsonblockkeyword_binding

Adds a JSON block-keyword relaxation (or rule) to an application firewall profile. The application firewall inspects JSON payloads for the configured keyword at the given JSON key and URL; this binding lets you control how that keyword is treated (for example, flag it as a regex match, or block requests containing it) so you can tune protection for a specific API or JSON endpoint without weakening the rest of the profile.


## Example usage

```hcl
resource "citrixadc_appfwprofile" "tf_appfwprofile" {
  name                     = "tf_appfwprofile"
  bufferoverflowaction     = ["none"]
  contenttypeaction        = ["none"]
  cookieconsistencyaction  = ["none"]
  creditcard               = ["none"]
  creditcardaction         = ["none"]
  crosssitescriptingaction = ["none"]
  csrftagaction            = ["none"]
  denyurlaction            = ["none"]
  dynamiclearning          = ["none"]
  fieldconsistencyaction   = ["none"]
  fieldformataction        = ["none"]
  fileuploadtypesaction    = ["none"]
  inspectcontenttypes      = ["none"]
  jsondosaction            = ["none"]
  jsonsqlinjectionaction   = ["none"]
  jsonxssaction            = ["none"]
  multipleheaderaction     = ["none"]
  sqlinjectionaction       = ["none"]
  starturlaction           = ["none"]
  type                     = ["HTML"]
  xmlattachmentaction      = ["none"]
  xmldosaction             = ["none"]
  xmlformataction          = ["none"]
  xmlsoapfaultaction       = ["none"]
  xmlsqlinjectionaction    = ["none"]
  xmlvalidationaction      = ["none"]
  xmlwsiaction             = ["none"]
  xmlxssaction             = ["none"]
}

resource "citrixadc_appfwprofile_jsonblockkeyword_binding" "tf_binding" {
  name                         = citrixadc_appfwprofile.tf_appfwprofile.name
  jsonblockkeyword             = "passwd"
  keyname_json_blockkeyword    = "user.credentials"
  jsonblockkeywordurl          = "/api/v1/login"
  iskeyregex_json_blockkeyword = "NOTREGEX"
  jsonblockkeywordtype         = "literal"
  state                        = "ENABLED"
  comment                      = "Block the passwd keyword on the login endpoint"
}
```


## Argument Reference

* `name` - (Required) Name of the profile to which to bind an exemption or rule. Changing this forces a new resource to be created.
* `jsonblockkeyword` - (Required) Field name of the JSON block keyword binding. Changing this forces a new resource to be created.
* `keyname_json_blockkeyword` - (Required) JSON block keyword keyname (the JSON key under which the keyword is matched). Changing this forces a new resource to be created.
* `jsonblockkeywordurl` - (Required) The JSON block keyword URL on which the keyword is inspected. Changing this forces a new resource to be created.
* `iskeyregex_json_blockkeyword` - (Optional) Is the JSON block keyword key a regular expression? Defaults to `"NOTREGEX"`. Possible values: [ REGEX, NOTREGEX ]
* `jsonblockkeywordtype` - (Optional) JSON block keyword type. Possible values: [ literal, PCRE ]
* `state` - (Optional) Enable or disable the binding. Possible values: [ ENABLED, DISABLED ]
* `comment` - (Optional) Any comments about the purpose of the profile, or other useful information about the profile.
* `isautodeployed` - (Optional) Is the rule auto deployed by a dynamic profile?

## Attribute Reference

In addition to the arguments, the following attributes are available:

* `id` - The ID of the appfwprofile_jsonblockkeyword_binding resource. It is a composite of comma-separated `key:value` pairs in the order `jsonblockkeyword:<value>,jsonblockkeywordurl:<value>,keyname_json_blockkeyword:<value>,name:<value>`.
* `alertonly` - (Read-only) Send SNMP alert? This value is server-managed and populated from the Citrix ADC; it is not user-settable.
* `resourceid` - (Read-only) A server-assigned identifier that identifies the rule. This value is generated by the Citrix ADC and populated into state on read; it is not user-settable.


## Import

An appfwprofile_jsonblockkeyword_binding can be imported using its id, e.g.

```shell
terraform import citrixadc_appfwprofile_jsonblockkeyword_binding.tf_binding "jsonblockkeyword:passwd,jsonblockkeywordurl:/api/v1/login,keyname_json_blockkeyword:user.credentials,name:tf_appfwprofile"
```
