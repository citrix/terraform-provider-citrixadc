# Resource: gslbconfig_sync

Synchronizes the Global Server Load Balancing (GSLB) configuration across all GSLB sites in the deployment. Applying this resource triggers an imperative `sync` action on the master node, which propagates the master's GSLB configuration to the slave nodes at the other sites. It does not manage any persistent object on the Citrix ADC: there is no NITRO GET, update, or delete endpoint for this action, so the Read, Update, and Delete operations are state-only no-ops. A successful apply records a synthetic ID in Terraform state and never reads anything back.

Use this resource when you want a GSLB configuration sync to run as part of a Terraform apply. A working GSLB deployment with the GSLB sites already configured (using `citrixadc_gslbsite` and related resources) is a prerequisite; this resource only synchronizes that existing configuration.


## Example usage

A bare sync (synchronizes the GSLB sites with the master node's configuration using default behavior):

```hcl
resource "citrixadc_gslbconfig_sync" "tf_gslbconfig_sync" {
}
```

A sync that forces synchronization of all sites and emits verbose output:

```hcl
resource "citrixadc_gslbconfig_sync" "tf_gslbconfig_sync" {
  forcesync = "all-sites"
  debug     = true
}
```


## Argument Reference

The following flags are mutually exclusive within the group `preview` / `forcesync` / `command` / `nowarn` / `saveconfig` — specify at most one of them. The `debug` flag is independent and can be combined with any of the others.

* `preview` - (Optional) Do not synchronize the GSLB sites, but display the commands that would be applied on the slave node upon synchronization. Mutually exclusive with the Save Configuration option.
* `forcesync` - (Optional) Force synchronization of the specified site even if a dependent configuration on the remote site is preventing synchronization or if one or more GSLB entities on the remote site have the same name but are of a different type. You can specify either the name of the remote site that you want to synchronize with the local site, or you can specify the string `all-sites` to synchronize all the sites in the GSLB setup with the site on the master node. Note: If you select the Force Sync option, the synchronization starts without displaying the commands that are going to be executed.
* `command` - (Optional) Run the specified command on the master node and then on all the slave nodes. You cannot use this option with the force sync and preview options.
* `nowarn` - (Optional) Suppress the warning and the confirmation prompt that are displayed before site synchronization begins. This option can be used in automation scripts that must not be interrupted by a prompt.
* `saveconfig` - (Optional) Save the configuration on all the nodes participating in the synchronization process, automatically. The master saves its configuration immediately before synchronization begins. Slave nodes save their configurations after the process of synchronization is complete. A slave node saves its configuration only if the configuration difference was successfully applied to it. Mutually exclusive with the Preview option.
* `debug` - (Optional) Generate verbose output when synchronizing the GSLB sites. The Debug option generates more verbose output than the sync gslb config command in which the option is not used, and is useful for analyzing synchronization issues.


## Attribute Reference

In addition to the arguments, the following attributes are available:

* `id` - A synthetic, constant string `"gslbconfig_sync"`. This action-only resource has no Citrix ADC identity, so the ID is generated by the provider and does not correspond to any server-side object.
