systemsession

Configuration for system session resource.

Read/write properties

sid<Double>

ID of the system session about which to display information.
Minimum value = 1

all<Boolean>

Terminate all the system sessions except the current session.

Read only properties

username<String>

user name of the session.

logintime<String>

logged-in time of this session.

logintimelocal<String>

logged-in time (in local ADC timezone) of this session.

lastactivitytime<String>

last activity time of this session.

lastactivitytimelocal<String>

last activity time (in local ADC timezone) of this session.

expirytime<Integer>

Time left in expire the session in seconds.

numofconnections<Double>

number of connection using this token.

currentconn<Boolean>

True if the token is used for current session.

clienttype<String>

client type of the session.
Possible values = CLI, CONSOLE, API, GUI

partitionname<String>

Name of the partition where the user is currently present.

clientipaddress<String>

IP address of client connecting to ADC.

_nextgenapiresource<String>

.

__count<Double>

count parameter

Operations

You can perform the following operations:

get (all)getcountkill

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
