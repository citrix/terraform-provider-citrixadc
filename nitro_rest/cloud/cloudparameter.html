cloudparameter

Configuration for cloud parameter resource.

Read/write properties

controllerfqdn<String>

FQDN of the controller to which the Citrix ADC SDProxy Connects.
Minimum length = 1

controllerport<Integer>

Port number of the controller to which the Citrix ADC SDProxy connects.
Minimum value = 1
Range 1 - 65535
* in CLI is represented as 65535 in NITRO API

instanceid<String>

Instance ID of the customer provided by Trust.
Minimum length = 1

customerid<String>

Customer ID of the citrix cloud customer.
Minimum length = 1

resourcelocation<String>

Resource Location of the customer provided by Trust.
Minimum length = 1

activationcode<String>

Activation code for the NGS Connector instance.

deployment<String>

Describes if the customer is a Staging/Production or Dev Citrix Cloud customer.
Possible values = Production, Staging, Dev

connectorresidence<String>

Identifies whether the connector is located Onprem, Aws or Azure.
Possible values = None, Onprem, Aws, Azure, Cpx

Read only properties

controlconnectionstatus<String>

Status of the control connection: In case of successful registration to the controller, connection status will be shown as "Registered" else "Unregistered".

_nextgenapiresource<String>

.

Operations

You can perform the following operations:

updateunsetget (all)

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>

HTTP Method:PUT

Request Headers:

Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
Content-Type:application/json

Request Payload:

{"cloudparameter":{
      "controllerfqdn":<String_value>,
      "controllerport":<Integer_value>,
      "instanceid":<String_value>,
      "customerid":<String_value>,
      "resourcelocation":<String_value>,
      "activationcode":<String_value>,
      "deployment":<String_value>,
      "connectorresidence":<String_value>
}}

Response:

HTTP Status Code on Success: 200 OK
HTTP Status Code on Failure: 4xx <string> (for general HTTP errors) or 5xx <string> (for NetScaler-specific errors). The response payload provides details of the error
