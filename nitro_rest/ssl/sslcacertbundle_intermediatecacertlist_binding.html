sslcacertbundle_intermediatecacertlist_binding

Binding object showing the intermediatecacertlist that can be bound to sslcacertbundle.

Read/write properties

subject<String>

Subject name.

cacertbundlename<String>

Name given to the CA certbundle. The name will be used for bind/unbind/update operations. Must begin with an ASCII alphanumeric or underscore (_) character, and must contain only ASCII alphanumeric, underscore, hash (#), period (.), space, colon (:), at (@), equals (=), and hyphen (-) characters. The following requirement applies only to the Citrix ADC CLI: If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, "my file" or 'my file').
Minimum length = 1
Maximum length = 127

Read only properties

status<String>

Status of the certificate.
Possible values = Valid, Not yet valid, Expired

signaturealg<String>

Signature algorithm.

serial<String>

Serial number.

sandns<String>

Subject Alternative Name (SAN) is an extension to X.509 that allows various values to be associated with a security certificate using a subjectAltName field. These values are called "Subject Alternative Names" (SAN). This field is for DNS names.

issuer<String>

Issuer name.

publickeysize<Double>

Size of the public key.

daystoexpiration<Double>

Days remaining for the certificate to expire.

sanipadd<String>

Subject Alternative Name (SAN) is an extension to X.509 that allows various values to be associated with a security certificate using a subjectAltName field. These values are called "Subject Alternative Names" (SAN). This field is for IP address.

clientcertnotbefore<String>

Not-Before date.

clientcertnotafter<String>

Not-After date.

publickey<String>

Public key algorithm.

__count<Double>

count parameter

    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
