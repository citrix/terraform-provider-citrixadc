ssldtlsprofile

Configuration for DTLS profile resource.

Read/write properties

name<String>

Name for the DTLS profile. Must begin with an ASCII alphanumeric or underscore (_) character, and must contain only ASCII alphanumeric, underscore, hash (#), period (.), space, colon (:), at (@),equals sign (=), and hyphen (-) characters. Cannot be changed after the profile is created.
Minimum length = 1
Maximum length = 127

pmtudiscovery<String>

Source for the maximum record size value. If ENABLED, the value is taken from the PMTU table. If DISABLED, the value is taken from the profile.
Default value: DISABLED
Possible values = ENABLED, DISABLED

maxrecordsize<Double>

Maximum size of records that can be sent if PMTU is disabled.
Default value: 1459
Minimum value = 250
Maximum value = 1459

maxretrytime<Double>

Wait for the specified time, in seconds, before resending the request.
Default value: 3

helloverifyrequest<String>

Send a Hello Verify request to validate the client.
Default value: ENABLED
Possible values = ENABLED, DISABLED

terminatesession<String>

Terminate the session if the message authentication code (MAC) of the client and server do not match.
Default value: DISABLED
Possible values = ENABLED, DISABLED

maxpacketsize<Double>

Maximum number of packets to reassemble. This value helps protect against a fragmented packet attack.
Default value: 120
Minimum value = 0
Maximum value = 86400

maxholdqlen<Double>

Maximum number of datagrams that can be queued at DTLS layer for processing.
Default value: 32
Minimum value = 32
Maximum value = 65535

maxbadmacignorecount<Double>

Maximum number of bad MAC errors to ignore for a connection prior disconnect. Disabling parameter terminateSession terminates session immediately when bad MAC is detected in the connection.
Default value: 100
Minimum value = 1
Maximum value = 65535

initialretrytimeout<Double>

Initial time out value to retransmit the last flight sent from the NetScaler.
Default value: 3
Minimum value = 1
Maximum value = 3

Read only properties

builtin<String[]>

Flag to determine whether dtls profile is built-in or not.
Possible values = MODIFIABLE, DELETABLE, IMMUTABLE, PARTITION_ALL

feature<String>

The feature to be checked while applying this config.

_nextgenapiresource<String>

.

count parameter

Operations

You can perform the following operations:

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
