botprofile_ratelimit_binding

Binding object showing the ratelimit that can be bound to botprofile.

Read/write properties

logmessage<String>

Message to be logged for this binding.
Minimum length = 1

bot_rate_limit_enabled<String>

Enable or disable rate-limit binding.
Default value: OFF
Possible values = ON, OFF

bot_rate_limit_action<String[]>

One or more actions to be taken when the current rate becomes more than the configured rate. Only LOG action can be combined with DROP, REDIRECT, RESPOND_STATUS_TOO_MANY_REQUESTS or RESET action.
Default value: NONE
Possible values = NONE, LOG, DROP, REDIRECT, RESET, RESPOND_STATUS_TOO_MANY_REQUESTS

bot_ratelimit<Boolean>

Rate-limit binding. Maximum 30 bindings can be configured per profile for rate-limit detection. For SOURCE_IP type, only one binding can be configured, and for URL type, only one binding is allowed per URL, and for SESSION type, only one binding is allowed for a cookie name. To update the values of an existing binding, user has to first unbind that binding, and then needs to bind again with new values.

limittype<String>

Rate-Limiting traffic Type.
Default value: BURSTY
Possible values = BURSTY, SMOOTH

countrycode<String>

Country name which is used for geolocation rate-limiting.

bot_rate_limit_url<String>

URL for the resource based rate-limiting.

timeslice<Double>

Time interval during which requests are tracked to check if they cross the given rate.
Default value: 1000
Minimum value = 10

condition<String>

Expression to be used in a rate-limiting condition. This expression result must be a boolean value.

name<String>

Name for the profile. Must begin with a letter, number, or the underscore character (_), and must contain only letters, numbers, and the hyphen (-), period (.), pound (#), space ( ), at (@), equals (=), colon (:), and underscore (_) characters. Cannot be changed after the profile is added. The following requirement applies only to the Citrix ADC CLI: If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, "my profile" or 'my profile').
Minimum length = 1
Maximum length = 31

cookiename<String>

Cookie name which is used to identify the session for session rate-limiting.

bot_rate_limit_type<String>

Rate-limiting type Following rate-limiting types are allowed: *SOURCE_IP - Rate-limiting based on the client IP. *SESSION - Rate-limiting based on the configured cookie name. *URL - Rate-limiting based on the configured URL. *GEOLOCATION - Rate-limiting based on the configured country name. *JA3_FINGERPRINT - Rate-limiting based on client SSL JA3 fingerprint.
Possible values = SESSION, SOURCE_IP, URL, GEOLOCATION, JA3_FINGERPRINT

bot_bind_comment<String>

Any comments about this binding.
Minimum length = 1

rate<Double>

Maximum number of requests that are allowed in this session in the given period time.
Default value: 1
Minimum value = 1

__count<Double>

count parameter

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
