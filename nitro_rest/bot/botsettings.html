botsettings

Configuration for Bot engine settings resource.

Read/write properties

defaultprofile<String>

Profile to use when a connection does not match any policy. Default setting is " ", which sends unmatched connections back to the Citrix ADC without attempting to filter them further.
Minimum length = 1

defaultnonintrusiveprofile<String>

Profile to use when the feature is not enabled but feature is licensed. NonIntrusive checks will be disabled and IPRep cronjob(24 Hours) will be removed if this is set to BOT_BYPASS.
Default value: BOT_STATS
Minimum length = 1
Possible values = BOT_BYPASS, BOT_STATS, BOT_LOG

javascriptname<String>

Name of the JavaScript that the Bot Management feature uses in response.
Must begin with a letter or number, and can consist of from 1 to 31 letters, numbers, and the hyphen (-) and underscore (_) symbols.

The following requirement applies only to the Citrix ADC CLI:
If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, "my cookie name" or 'my cookie name').
Minimum length = 1

sessiontimeout<Double>

Timeout, in seconds, after which a user session is terminated.
Minimum value = 1
Maximum value = 65535

sessioncookiename<String>

Name of the SessionCookie that the Bot Management feature uses for tracking.
Must begin with a letter or number, and can consist of from 1 to 31 letters, numbers, and the hyphen (-) and underscore (_) symbols.

The following requirement applies only to the Citrix ADC CLI:
If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, "my cookie name" or 'my cookie name').
Minimum length = 1

dfprequestlimit<Double>

Number of requests to allow without bot session cookie if device fingerprint is enabled.
Minimum value = 1

signatureautoupdate<String>

Flag used to enable/disable bot auto update signatures.
Default value: OFF
Possible values = ON, OFF

signatureurl<String>

URL to download the bot signature mapping file from server.
Default value: https://nsbotsignatures.s3.amazonaws.com/BotSignatureMapping.json

proxyserver<String>

Proxy Server IP to get updated signatures from AWS.

proxyport<Integer>

Proxy Server Port to get updated signatures from AWS.
Default value: 8080
Range 1 - 65535
* in CLI is represented as 65535 in NITRO API

trapurlautogenerate<String>

Enable/disable trap URL auto generation. When enabled, trap URL is updated within the configured interval.
Default value: OFF
Possible values = ON, OFF

trapurlinterval<Double>

Time in seconds after which trap URL is updated.
Default value: 3600
Minimum value = 300
Maximum value = 86400

trapurllength<Double>

Length of the auto-generated trap URL.
Default value: 32
Minimum value = 10
Maximum value = 255

proxyusername<String>

Proxy Username.
Minimum length = 1

proxypassword<String>

Password with which user logs on.
Minimum length = 1

Read only properties

builtin<String[]>

Flag to determine if bot engine setting is built-in or not.
Possible values = MODIFIABLE, DELETABLE, IMMUTABLE, PARTITION_ALL

feature<String>

The feature to be checked while applying this config.

_nextgenapiresource<String>

.

Operations

You can perform the following operations:

updateunsetget (all)

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>

NOTE: Mandatory parameters are marked red and bold. Placeholder text is marked in green.
