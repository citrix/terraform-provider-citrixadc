servicegroup_servicegroupentitymonbindings_binding

Binding object showing the servicegroupentitymonbindings that can be bound to servicegroup.

Read/write properties

port<Integer>

Port number of the service. Each service must have a unique port number.
Range 1 - 65535
* in CLI is represented as 65535 in NITRO API

passive<Boolean>

Indicates if load monitor is passive. A passive load monitor does not remove service from LB decision when threshold is breached.

servicegroupname<String>

Name of the service group.
Minimum length = 1

nameserver<String>

Specify the nameserver to which the query for bound domain needs to be sent. If not specified, use the global nameserver.

servicegroupentname2<String>

.

serverid<Double>

The identifier for the service. This is used when the persistency type is set to Custom Server ID.

weight<Double>

.
Default value: 1
Minimum value = 1
Maximum value = 100

dbsttl<Double>

Specify the TTL for DNS record for domain based service.The default value of ttl is 0 which indicates to use the TTL received in DNS response for monitors.
Default value: 0

order<Double>

Order number to be assigned to the servicegroup member.
Minimum value = 1
Maximum value = 8192

monitor_name<String>

Monitor name.

state<String>

Initial state of the service after binding.
Default value: ENABLED
Possible values = ENABLED, DISABLED

customserverid<String>

Unique service identifier. Used when the persistency type for the virtual server is set to Custom Server ID.
Default value: "None"

hashid<Double>

Unique numerical identifier used by hash based load balancing methods to identify a service.
Minimum value = 1

Read only properties

monitorcurrentfailedprobes<Double>

Total number of currently failed probes.

responsetime<Double>

Response time of this monitor in milli sec.

monitor_state<String>

The running state of the monitor on this service.
Possible values = UP, DOWN, UNKNOWN, BUSY, OUT OF SERVICE, GOING OUT OF SERVICE, DOWN WHEN GOING OUT OF SERVICE, NS_EMPTY_STR, Unknown, DISABLED

monitortotalfailedprobes<Double>

Total number of failed probes.

lastresponse<String>

The string form of monstatcode.

monitortotalprobes<Double>

Total number of probes sent to monitor this service.

__count<Double>

count parameter

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
