routeMap

Configuration for routeMap resource.

Read/write properties

name<String>

Route map tag.

action<String>

Specifies if the route-map denies or permits the operations.
Possible values: permit, deny

localPreference<Integer>

Set the BGP local preference path attribute.
Minimum value: 0
Maximum value: 4294967295

matchAsPath<String>

Match the BGP AS-path list.

matchCommunity<String>

Match BGP community list.

matchIpAddress<String>

Match IP address of route. IP access-list number in the range <1-99> or acces-list name.

matchIpNextHop<String>

Match next-hop address of route. IP access-list number in the range <1-99> or acces-list name.

matchMetric<Integer>

Match values from routing table. Match metric of route.
Minimum value: 0
Maximum value: 4294967295

matchRouteType<String>

Match OSPF external routes of type 1 or type 2 metrics.
Possible values: type-1, type-2

sequence<Integer>

Sequence to insert to/delete from existing route-map entry.
Minimum value: 1
Maximum value: 65535

setAsPath<String>

Set the prepend string for a BGP AS-path attribute

setCommunity<String>

Set the BGP community attribute.

setIpNextHop<String>

Set next hop address of a route.

setMetric<Integer>

Set metric value for destination routing protocol.
Minimum value: 0
Maximum value: 4294967295

setMetricType<String>

Set type of metric for destination routing protocol. OSPF external type 1 metric or OSPF external type 2 metric.
Possible values: type-1, type-2

weight<Integer>

Set BGP weight for routing table.
Minimum value: 0
Maximum value: 4294967295

Operations

You can perform the following operations:

addsetunsetdeleteget

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>

NOTE: Mandatory parameters are marked red and bold. Placeholder text is marked in green.
