ospfInterface

Configuration for ospfInterface resource.

Read/write properties

authKey<String>

Authentication password (key).

authType<String>

Authentication type on the ospf interface.
Possible values: null, simple, message-digest

bfd<Boolean>

Enable bfd on interface

cost<Integer>

Interface cost.
Minimum value: 1
Maximum value: 65535

deadInterval<Integer>

Interval after which a neighbor is declared dead
Minimum value: 1
Maximum value: 65535

helloInterval<Integer>

Time between HELLO packets
Minimum value: 1
Maximum value: 65535

mtu<Integer>

OSPF interface MTU.
Minimum value: 576
Maximum value: 65535

name<String>

Name of the interface

networkType<String>

Network type.
Possible values: broadcast, non-broadcast, point-to-multipoint, point-to-point

priority<Integer>

Router priority.
Minimum value: 0
Maximum value: 255

retransmitInterval<Integer>

Time between retransmitting lost link state advertisements.
Minimum value: 1
Maximum value: 65535

transmitDelay<Integer>

Link state transmit delay.
Minimum value: 1
Maximum value: 65535

Operations

You can perform the following operations:

addsetunsetget

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
