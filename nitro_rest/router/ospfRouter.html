ospfRouter

Configuration for ospfRouter resource.

Read/write properties

area<Integer>

OSPF area ID as a decimal value.
Minimum value: 0
Maximum value: 4294967295

ipaddress<String>

OSPF network prefix.

netmask<Integer>

The netmask length
Minimum value: 0
Maximum value: 32

passiveInterface[<String>]

Suppress routing updates on an interface

processId<Integer>

OSPF process ID
Minimum value: 1
Maximum value: 65535

metric<Integer>

The metric of redistributed routes.
Minimum value: 0
Maximum value: 16777214

metricType<Integer>

OSPF metric type for default routes - OSPF External Type 1 metrics or Type 2 metrics.
Minimum value: 1
Maximum value: 2

ospfProcessId<Integer>

The process ID of the ospf instance to redistribute the routes from.

protocol<String>

The protocol from which routes need to be redistributed.
Possible values: bgp, connected, intranet, isis, kernel, ospf, rip, static

routeMap<String>

Route map reference

tag<Integer>

Set tag for routes redistributed into OSPF
Minimum value: 0
Maximum value: 4294967295

routerId<String>

Router-id for the OSPF process

Operations

You can perform the following operations:

addsetunsetdeleteget

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
