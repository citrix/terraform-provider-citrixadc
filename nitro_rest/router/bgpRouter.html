bgpRouter

Configuration for bgpRouter resource.

Read/write properties

addressFamily<String>

Address family of the routes.
Possible values: ipv4, ipv6

protocol<String>

The protocol from which routes need to be redistributed.
Possible values: kernel, connected, static, rip, ospf, isis, intranet

routeMap<String>

Route map reference

localAS<Integer>

Autonomous system number.
Minimum value: 1
Maximum value: 4294967295

ASOriginationInterval<Integer>

Minimum interval between sending AS-origination routing updates.
Minimum value: 1
Maximum value: 65535

address<String>

The ipv4 or ipv6 address of the neighboring router.

advertisementInterval<Integer>

Minimum interval between sending BGP routing updates.
Minimum value: 0
Maximum value: 65535

activate<Boolean>

Enable the Address Family for the neighbor.

addressFamily<String>

Address family identifier
Possible values: ipv4, ipv6

direction<String>

Apply the route-map to incoming routes or outgoing routes.
Possible values: in, out

name<String>

Name of the route-map.

connectTimer<Integer>

Time interval (in seconds) for the ConnectRetry timer.
Minimum value: 1
Maximum value: 65535

holdTimer<Integer>

Holdtime for the neighbor.

holdTimerConfig<Integer>

Configured hold time for the neighbor.
Minimum value: 0
Maximum value: 65535

keepaliveTimer<Integer>

Keepalive time for the neighbor.

keepaliveTimerConfig<Integer>

Configured keepalive time for the neighbor.
Minimum value: 0
Maximum value: 65535

localAddress<String>

The local IP address of this entry's BGP connection.

localPort<Integer>

The local port of the TCP connection between the BGP peers.

md5Password<String>

MD5 password for the neighbor.

multihopBfd<Boolean>

Enable BFD for multihop session

remoteAS<Integer>

AS number of the neighbor.
Minimum value: 1
Maximum value: 4294967295

remotePort<Integer>

The remote port of the TCP connection between the BGP peers.

singlehopBfd<Boolean>

Enable BFD on this neighbor

state<String>

The neighbor connection state.

updateSource<String>

Source of routing updates.

routerId<String>

The router identifier for the bgp routing instance.

Operations

You can perform the following operations:

addsetunsetdeleteget

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>

NOTE: Mandatory parameters are marked red and bold. Placeholder text is marked in green.
