nsmigration

Configuration for Migration operation resource.

Read/write properties

dumpsession<String>

Displays the current active migrated session details, if DUMPSESSION option is YES.
Default value: NO
Possible values = YES, NO

Read only properties

migrationstatus<String>

ISSU Migration Status.
Possible values = Migration is not yet started, Migration is in progress and Failover is not yet done, Migration is in progress and Failover is completed, Rollback is initiated and Failover is not yet done, Rollback is initiated and Failover is completed, Migration is completed

migrationstarttime<String>

Timestamp for start migration.
Minimum length = 1

migrationendtime<String>

Timestamp for migration complete.
Minimum length = 1

migrationrollbackstarttime<String>

Timestamp for start migration rollback.
Minimum length = 1

srcip<String>

IP address to match against the source IP address of an incoming IPv4 packet.

srcport<Double>

Port number to match against the source port number of an incoming IPv4 packet.

destip<String>

IP address to match against the destination IP address of an incoming IPv4 packet.

destport<Double>

Port number to match against the destination port number of an incoming IPv4 packet.

timeout<Double>

Time for which the connection is idle.

migdfdsessionsallocated<Double>

Number of migration sessions allocated.

migdfdsessionsactive<Double>

Number of migration sessions active.

migl4sessionsallocated<Double>

Number of migration sessions allocated.

migl4sessionsactive<Double>

Number of migration sessions active.

migdfdsessionsallocatedrollback<Double>

Number of migration sessions allocated.

migdfdsessionsactiverollback<Double>

Number of migration sessions active.

migl4sessionsallocatedrollback<Double>

Number of migration sessions allocated.

migl4sessionsactiverollback<Double>

Number of migration sessions active.

mighastateflag<Double>

Current master state in HA.

Operations

You can perform the following operations:

get (all)count

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
