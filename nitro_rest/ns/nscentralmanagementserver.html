nscentralmanagementserver

Configuration for centralmanagementserver resource.

Read/write properties

type<String>

Type of the central management server. Must be either CLOUD or ONPREM depending on whether the server is on the cloud or on premise.
Possible values = CLOUD, ONPREM

username<String>

Username for access to central management server. Must begin with a letter, number, or the underscore character (_), and must contain only letters, numbers, and the hyphen (-), period (.) pound (#), space ( ), at (@), equals (=), colon (:), and underscore characters.

The following requirement applies only to the Citrix ADC CLI:
If the name includes one or more spaces, enclose the name in double or
single quotation marks (for example, "my ns centralmgmtserver" or "my ns centralmgmtserver").
Minimum length = 1
Maximum length = 127

password<String>

Password for access to central management server. Required for any user account.
Minimum length = 1
Maximum length = 127

activationcode<String>

Activation code is used to register to ADM service.
Minimum length = 1
Maximum length = 127

ipaddress<String>

Ip Address of central management server.
Minimum length = 1

servername<String>

Fully qualified domain name of the central management server or service-url to locate ADM service.

validatecert<String>

validate the server certificate for secure SSL connections.
Default value: YES
Possible values = YES, NO

deviceprofilename<String>

Device profile is created on ADM and contains the user name and password of the instance(s).
Minimum length = 1
Maximum length = 255

adcusername<String>

ADC username used to create device profile on ADM.
Minimum length = 1
Maximum length = 127

adcpassword<String>

ADC password used to create device profile on ADM.
Minimum length = 1
Maximum length = 127

Read only properties

instanceid<String>

Instance ID of the customer provided by Trust.

customerid<String>

Customer ID of the citrix cloud customer.

admserviceenvironment<String>

ADM service environment (PRODUCTION/STAGING/DEV).
Possible values = PRODUCTION, STAGING, DEV

admserviceconnectionstatus<String>

Built-in agent's (mastools) connection status to ADM service.
Possible values = CONNECTED, DISCONNECTED

_nextgenapiresource<String>

.

__count<Double>

count parameter

Operations

You can perform the following operations:

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
