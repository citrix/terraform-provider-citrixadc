nscapacity

Configuration for capacity resource.

Read/write properties

bandwidth<Double>

System bandwidth limit.

platform<String>

appliance platform type.
Possible values = VS10, VE10, VP10, VS25, VE25, VP25, VS50, VE50, VP50, VS200, VE200, VP200, VS1000, VE1000, VP1000, VS3000, VE3000, VP3000, VS5000, VE5000, VP5000, VS8000, VE8000, VP8000, VS10000, VE10000, VP10000, VS15000, VE15000, VP15000, VS25000, VE25000, VP25000, VS40000, VE40000, VP40000, VS100000, VE100000, VP100000, CP1000

vcpu<Boolean>

licensed using vcpu pool.

edition<String>

Product edition.
Possible values = Standard, Enterprise, Platinum

unit<String>

Bandwidth unit.
Possible values = Gbps, Mbps

username<String>

Username to authenticate with ADM Agent for LAS licensing. Must begin with a letter, number, or the underscore character (_), and must contain only letters, numbers, and the hyphen (-), period (.) pound (#), space ( ), at (@), equals (=), colon (:), and underscore characters.
Minimum length = 1
Maximum length = 127

password<String>

Password to use when authenticating with ADM Agent for LAS licensing.
Minimum length = 1

nodeid<Double>

Unique number that identifies the cluster node.
Minimum value = 0
Maximum value = 31

Read only properties

actualbandwidth<Double>

Bandwith in MBPS.

vcpucount<Double>

number of vCPUs licensed.
Minimum value = 1
Maximum value = 20

maxvcpucount<Double>

number of max vCPUs.
Minimum value = 1
Maximum value = 20

maxbandwidth<Double>

Maximum Bandwidth.

minbandwidth<Double>

Minimum Bandwidth.

instancecount<Double>

VPX will consume one instance and MPX will consume zero instance.

daystoexpiration<Double>

Days to expire.
Minimum value = 0
Maximum value = 4294967294

_nextgenapiresource<String>

.

Operations

You can perform the following operations:

updateunsetget (all)

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
