icaparameter

Configuration for Config Parameters for NS ICA resource.

Read/write properties

enablesronhafailover<String>

Enable/Disable Session Reliability on HA failover. The default value is No.
Default value: NO
Possible values = YES, NO

hdxinsightnonnsap<String>

Enable/Disable HDXInsight for Non NSAP ICA Sessions. The default value is Yes.
Default value: YES
Possible values = YES, NO

edtpmtuddf<String>

Enable/Disable DF enforcement for EDT PMTUD Control Blocks.
Default value: ENABLED
Possible values = ENABLED, DISABLED

edtpmtuddftimeout<Double>

DF enforcement timeout for EDTPMTUDDF.
Default value: 100
Minimum value = 10
Maximum value = 65535

l7latencyfrequency<Double>

Specify the time interval/period for which L7 Client Latency value is to be calculated. By default, L7 Client Latency is calculated for every packet. The default value is 0.
Default value: 0
Minimum value = 0
Maximum value = 60

edtlosstolerant<String>

Enable/Disable EDT Loss Tolerant feature.
Default value: ENABLED
Possible values = ENABLED, DISABLED

edtpmtudrediscovery<String>

Enable/Disable EDT PMTUD Rediscovery.
Default value: DISABLED
Possible values = ENABLED, DISABLED

dfpersistence<String>

Enable/Disable DF Persistence.
Default value: DISABLED
Possible values = ENABLED, DISABLED

Read only properties

builtin<String[]>

Indicates that the ICA parameter is a built-in (SYSTEM INTERNAL) type.
Possible values = MODIFIABLE, DELETABLE, IMMUTABLE, PARTITION_ALL

_nextgenapiresource<String>

.

Operations

You can perform the following operations:

updateunsetget (all)

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
