policypatsetfile

Configuration for patset file resource.

Read/write properties

src<String>

URL in protocol, host, path, and file name format from where the patset file will be imported. If file is already present, then it can be imported using local keyword (import patsetfile local:filename patsetfile1)
NOTE: The import fails if the object to be imported is on an HTTPS server that requires client certificate authentication for access.
Minimum length = 1
Maximum length = 2047

name<String>

Name to assign to the imported patset file. Unique name of the pattern set. Not case sensitive. Must begin with an ASCII letter or underscore (_) character and must contain only alphanumeric and underscore characters.
Minimum length = 1
Maximum length = 31

overwrite<Boolean>

Overwrites the existing file.

delimiter<String>

patset file patterns delimiter.
Default value: 10

charset<String>

Character set associated with the characters in the string.
Possible values = ASCII, UTF_8

comment<String>

Any comments to preserve information about this patsetfile.

imported<Boolean>

When set, display only shows all imported patsetfiles.
Default value: 0

Read only properties

totalpatterns<Double>

Total number of patterns in the patset file.

boundpatterns<Double>

Total number of patterns bound to a patset.

patsetname<String>

The patset with which the patsetfile is associated.

bindstatuscode<Double>

The status code of pattern bindings to patset.

bindstatus<String>

The status of pattern bindings to patset.

_nextgenapiresource<String>

.

__count<Double>

count parameter

Operations

You can perform the following operations:

Importadddeletechangeget (all)getcount

Some options that you can use for each operation:

  • Getting warnings in response: NITRO allows you to get warnings in an operation by specifying the "warning" query parameter as "yes". For example, to get warnings while connecting to the NetScaler appliance, the URL is as follows:
    http://<netscaler-ip-address>/nitro/v1/config/lbvserver?warning=yes

    If any, the warnings are displayed in the "X-NITRO-WARNING" header and the HTTP status code is 209.

  • Authenticated access for individual NITRO operations: One of the authentication mechanisms is OneTimelogin/Basic authentication. You can use the below two headers specifying username and password and send it along with the regular nitro request.

    X-NITRO-USER:<username>
    X-NITRO-PASS:<password>

    Note: In such cases, make sure that the request header DOES not include the following:
    Cookie:NITRO_AUTH_TOKEN=<tokenvalue>
